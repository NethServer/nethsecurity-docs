# NethSecurity update and support policy

NethSecurity provides two main update methods:

* **Software package updates**, distributed regularly and installed without replacing the entire system image.These updates do not require a system reboot
* **Version updates**, performed by installing a new NethSecurity system image. These updates include changes to the underlying platform, require a system reboot, and are normally released every **4 to 6 months**

Security fixes are delivered using the most appropriate update method, depending on the affected component, the impact of the vulnerability, and system stability requirements.

When a fix cannot be safely applied through a standard package update, installing a new system version may be required. In the event of critical or actively exploited vulnerabilities, a new system image may be released earlier than originally scheduled.

## NethSecurity version support levels

### Current version - full support

The current version corresponds to the latest stable published system image and is recommended for:

* new installations
* upgrades of existing systems
* access to all available security fixes
* access to full technical support

The current version receives all applicable software updates.

### Previous version - limited support

The stable version immediately preceding the current release remains under limited support until the next system image is published.

During this period, the previous version receives only compatible security updates. However, upgrading to the current version may be required to resolve certain issues, including bugs already fixed in the latest release or fixes that cannot be applied to the previous version.

The version immediately preceding the current version remains supported, with limited support, until the release of the version following the current one. Based on the usual image release schedule, this period is generally between **4 and 6 months**.

### Earlier versions - unsupported

Versions older than the previous version do not receive new software updates, security fixes, or backports.


## Requirements for Nethesis technical support

:::note

Nethesis technical support is a paid service available only for systems covered by a valid subscription. The support levels described in this section therefore apply only to customers with an active subscription.

:::

To qualify for **full support**, the system must:

* run the current version
* have all available software updates installed

Systems running the previous version are **still supported** under the conditions outlined in the relevant section.

Versions older than the previous version are not covered by standard support. An upgrade to a supported version may be required before further support activities can be provided.

To maintain appropriate levels of security, stability, and compatibility, both software package updates and new system versions must be installed regularly and within the applicable update window.
